Concepto Léxico y Tokens

Presentado por Kevin García · Clasificación y estructura de los tokens en compiladores

🔤

¿Qué es el Concepto Léxico?

Constituye la base de la primera fase de un compilador. Define el conjunto de reglas que determinan qué secuencias de caracteres son válidas dentro de un lenguaje de programación.

🧩

Lexemas

Los patrones formales —representados mediante expresiones regulares— permiten agrupar los caracteres del código fuente en unidades con significado lógico llamadas lexemas.

🏷️

Token vs Lexema vs Patrón

Token: la categoría. Lexema: el valor real. Patrón: la regla regex. Ej: Token=ID, Lexema="edad", Patrón=[a-zA-Z_][a-zA-Z0-9_]*

Ejemplo · Código: int x = 10;
int → KEYWORD x → IDENTIFIER = → OPERATOR 10 → LITERAL ; → DELIMITER

✏️ Ejercicios de Práctica

1

¿Cuál es el lexema del token KEYWORD en la sentencia: while (i < 10)?

✅ ¡Correcto! "while" es una palabra reservada cuyo lexema es exactamente la cadena "while".
❌ El lexema es el valor literal en el código. "while" es la palabra reservada.
2

¿Cuántos tokens tiene la sentencia: float y = 3.14;?

✅ ¡Correcto! Los 5 tokens son: float(KW) · y(ID) · =(OP) · 3.14(NUM) · ;(DELIM)
❌ Cuenta cada elemento con significado: float, y, =, 3.14, ; = 5 tokens.
3

¿Cuál de estas opciones es un ejemplo de LEXEMA?

✅ ¡Correcto! "miVariable" es el lexema: el valor real que aparece en el código fuente.
❌ El lexema es lo que literalmente escribes. Las otras opciones son tipos de token o patrones regex.

Definición e Implementación de Tokens

Presentado por Sofi Galvis y Fabián Jaimes · Tipos, anatomía y clasificación

🔑

Palabras Reservadas

Términos predefinidos con significado fijo. No pueden usarse como identificadores. Se verifican ANTES que cualquier otro patrón. Ej: if, while, int, return

🆔

Identificadores

Nombres definidos por el programador. Deben iniciar con letra o guion bajo. Se registran en la tabla de símbolos. Patrón: [a-zA-Z_][a-zA-Z0-9_]*

🔢

Literales / Constantes

Valores constantes en el código. Tipos: enteros (42), reales (3.14), cadenas ("hola"), booleanos (true/false). El token lleva el valor consigo.

➕

Operadores

Símbolos para operaciones: aritméticos (+,-,*,/), relacionales (==,!=), lógicos (&&,||,!). Regla Maximal Munch: == antes que =.

📍

Delimitadores

Estructuran y separan unidades del programa. No realizan operaciones. Ejemplos: ( ) { } [ ] ; ,

📊

Estructura Interna

Representación: <tipo, valor>
<KEYWORD,"int"> · <IDENTIFIER,"x"> · <OPERATOR,"="> · <LITERAL,10> · <DELIMITER,";">

📋 Tabla de Símbolos

  • Almacena nombre, tipo, ámbito y dirección de memoria de los identificadores
  • Se construye durante el análisis léxico y sintáctico
  • KEYWORD / OPERATOR / DELIMITER no requieren entrada en la tabla
  • Los identificadores siempre se registran con un puntero a la tabla

✏️ Ejercicios de Práctica

1

En la sentencia if (edad >= 18), ¿qué tipo de token es >=?

✅ ¡Correcto! ">=" es un operador relacional. Por Maximal Munch se reconoce como un solo token, no como ">"+"=".
❌ ">=" es un operador relacional (mayor o igual). No es keyword, delimitador ni identificador.
2

¿Cuál de los siguientes identificadores es INVÁLIDO en C/Java?

✅ ¡Correcto! Los identificadores no pueden comenzar con un dígito. El patrón exige letra o _ al inicio.
❌ El inválido es "2suma": inicia con número. Los identificadores deben comenzar con letra o guion bajo.
3

¿Qué tipo de token es "Hola Mundo" en el código?

✅ ¡Correcto! Texto entre comillas dobles = literal de cadena (STRING). Patrón: "[^"\n]*"
❌ Todo texto entre comillas dobles es un literal de cadena (STRING literal).

Expresiones Regulares

Presentado por Joel Steven Pineda Rincón · Aplicación en Analizadores Léxicos · USB 2026

∑

Definición Formal

Una ER es una notación algebraica para describir conjuntos de cadenas sobre un alfabeto Σ. Cada ER define un Lenguaje Regular reconocible por un Autómata Finito.

⚡

Propiedades Clave

Son el formalismo matemático base para los patrones léxicos. Se transforman mecánicamente en DFA que reconocen tokens en tiempo lineal O(n).

🔧

Herramientas: Lex/Flex

Generan automáticamente analizadores léxicos a partir de ER. Flujo: escribir .l → flex archivo.l → gcc lex.yy.c

🔄 Pipeline: De ER a Autómata Ejecutable

  • Paso 1: Expresión Regular → definición formal del patrón léxico
  • Paso 2: ER → NFA mediante el Algoritmo de Thompson
  • Paso 3: NFA → DFA mediante la construcción de subconjuntos
  • Paso 4: DFA → DFA Minimizado con el Algoritmo de Hopcroft
  • Resultado: Scanner determinista en tiempo lineal O(n)

✏️ Ejercicios de Práctica

1

¿Cuál ER reconoce correctamente un identificador válido en C?

✅ ¡Correcto! Comienza con letra o _, luego permite letras, dígitos o _ en cualquier cantidad.
❌ La ER correcta es [a-zA-Z_][a-zA-Z0-9_]* — debe iniciar con letra o _ y puede incluir dígitos después.
2

El operador * en una ER significa:

✅ ¡Correcto! Cierre de Kleene (*) = 0 o más. "Una o más" es +. "Cero o una" es ?
❌ El * (Cierre de Kleene) significa 0 o más repeticiones. El + es una o más, ? es cero o una.
3

Ante la entrada "==", ¿qué principio aplica el analizador léxico?

✅ ¡Correcto! Maximal Munch: consume la secuencia más larga posible → un solo token "==".
❌ Maximal Munch hace que "==" sea un único token de igualdad, no dos tokens "=".

Analizador Léxico

Presentado por Fabián Jaimes · Función de Análisis Léxico en los Compiladores

📖

Leer caracteres

El scanner lee el código fuente carácter por carácter, siendo la primera fase del compilador.

🧩

Agrupar en tokens

Agrupa secuencias en tokens con significado. Elimina espacios en blanco, comentarios y caracteres irrelevantes para la gramática.

🔍

Detectar errores

Identifica errores léxicos cuando encuentra caracteres que no coinciden con ningún patrón definido del lenguaje.

➡️

Entregar al sintáctico

Provee la secuencia de tokens al analizador sintáctico, actuando como interfaz entre el texto crudo y las fases superiores.

🚨 Errores Léxicos Comunes

  • Símbolo inválido: int x = 10 @; — el @ no pertenece al lenguaje
  • ID mal formado: int 1variable = 5; — no puede iniciar con número
  • Número inválido: float x = 12.3.4; — múltiples puntos decimales
  • Cadena sin cerrar: printf("Hola mundo); — falta la comilla de cierre
  • Carácter especial: int #valor = 10; — # no es válido en un identificador

✏️ Ejercicios de Práctica

1

¿Cuál de las siguientes sentencias contiene un ERROR LÉXICO?

✅ ¡Correcto! "3y" es inválido: no puede comenzar con un dígito. El analizador léxico lo detectaría como error.
❌ "float 3y = 2.5;" contiene el error léxico: "3y" inicia con número.
2

¿Qué hace el analizador léxico con los espacios en blanco y comentarios?

✅ ¡Correcto! El analizador léxico elimina espacios, tabuladores y comentarios. No aportan información semántica.
❌ Los espacios y comentarios son descartados. No tienen significado en la gramática del lenguaje.
3

¿El analizador léxico es la primera, segunda o tercera fase del compilador?

✅ ¡Correcto! El análisis léxico es SIEMPRE la primera fase. Transforma el texto fuente en tokens.
❌ El análisis léxico es la primera fase del compilador, siempre, sin excepción.
